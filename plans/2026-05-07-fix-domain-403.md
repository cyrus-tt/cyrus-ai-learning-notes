## 1. 一句话任务

用 Wrangler CLI 把 cyrustyj.xyz 从 Worker 代理切换到 Pages 原生 Custom Domain，消除 403。

## 2. 为什么做（Why）

`edge-proxy.js` Worker 是多余的反向代理层，导致 Zone 级安全规则（Bot Fight Mode）拦截请求返回 403。Pages 原生 Custom Domain 不经过 Zone 安全规则，无此问题。（见 lessons-learned.md L001）

## 3. 边界（不做什么）

- 不改网站代码/功能
- 不改 DNS nameserver（已在 Cloudflare）
- 不动 Pages Functions（`functions/api/`）

## 4. 方案步骤

1. `wrangler pages project list` — 确认 Pages 项目名
2. `wrangler pages domain list <project>` — 看当前已绑域名
3. `wrangler pages domain create <project> cyrustyj.xyz` — 绑 Pages Custom Domain
4. 确认 DNS CNAME 记录自动创建
5. `wrangler delete edge-proxy` — 删除 Worker（确认名称后执行）
6. `rm edge-proxy.js` — 删除仓库中的代理文件
7. `curl -sI https://cyrustyj.xyz` — 验证返回 200

## 5. 涉及文件/资源

- 删除：`edge-proxy.js`
- 外部：Cloudflare Dashboard（仅 Wrangler CLI 操作）
- CLI：`wrangler` 4.66.0

## 7. 风险 / 阻塞

- 风险：SSL 证书签发需要几分钟，期间可能短暂不可用
- 风险：DNS CNAME 可能与现有 A/AAAA 记录冲突，需先删旧记录
- 阻塞：Wrangler 需要已登录 Cloudflare 账号（`wrangler whoami`）

## 8. 回滚方案

如果 Pages Custom Domain 绑定失败：
1. 重新部署 Worker：`wrangler deploy edge-proxy.js --name edge-proxy`
2. 恢复 Worker Route：`wrangler route create cyrustyj.xyz/* --zone <zone-id>`
3. `edge-proxy.js` 在 git 里有历史，可 `git checkout` 恢复
